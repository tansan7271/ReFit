# Minidis 캐릭터 API 명세서

> **대상**: 백엔드 개발자  
> **목적**: 픽셀 캐릭터 상태 결정에 필요한 API 응답 스펙 정의  
> **최종 수정**: 2026-06-08  
> **프론트엔드 모듈 문서**: [frontend-module.md](./frontend-module.md)

---

## 개요

캐릭터 상태는 **네 가지 건강 기둥**의 점수를 기반으로 결정됩니다.

| 기둥 | 의미 | 낮을 때 캐릭터 상태 |
|------|------|-------------------|
| `sleep` | 수면 → 에너지 | `tired` (수면 부족) |
| `exercise` | 운동 → 성장 | `sedentary` (운동 부족) |
| `diet` | 식단 → 영양 | `overfed` (식단 불균형) |
| `routine` | 루틴 → 행복 | `derailed` (루틴 깨짐) |

---

## 엔드포인트

### `GET /character`

로그인한 유저의 캐릭터 상태 데이터를 반환합니다.

#### Response `200 OK`

```json
{
  "stats": {
    "sleep":    75,
    "exercise": 60,
    "diet":     80,
    "routine":  55
  },
  "workout_status":  "none",
  "is_sleeping":     false,
  "character_level": 3,
  "character_xp":    240
}
```

## 필드 명세

### `stats` 객체

네 기둥 각각의 오늘 하루 점수. **모든 필드 필수.**

| 필드 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `sleep` | `number` | 0–100 | 오늘 수면의 질·양 종합 점수 |
| `exercise` | `number` | 0–100 | 오늘 운동량 달성도 |
| `diet` | `number` | 0–100 | 오늘 식단 균형 점수 (과식/절식 모두 감점) |
| `routine` | `number` | 0–100 | 루틴(목표 습관) 연속 달성도 |

**점수 계산 가이드 (백엔드 재량):**
- 수면: 권장 수면 7–8h 기준, 시간·수면 효율·중간 기상 횟수 반영
- 운동: 목표 운동 시간 대비 달성률
- 식단: 칼로리·영양소 균형, 과식(-) / 결식(-) 모두 반영
- 루틴: 오늘 달성한 루틴 항목 수 / 전체 루틴 항목 수 × 100

---

### `workout_status`

| 값 | 의미 |
|----|------|
| `"none"` | 오늘 운동 없음 또는 완료 후 시간 경과 |
| `"active"` | **현재 운동 세션 진행 중** → 캐릭터 `workout` 상태 |
| `"completed"` | **방금 운동 완료** (완료 후 30분 이내 권장) → 캐릭터 `flex` 상태 |

---

### `is_sleeping`

| 값 | 의미 |
|----|------|
| `true` | **현재 수면 중** (취침 감지) → 캐릭터 `sleeping` 상태 |
| `false` | 깨어 있음 |

---

### `character_level` / `character_xp`

| 필드 | 타입 | 설명 |
|------|------|------|
| `character_level` | `number` (1–5) | 캐릭터 성장 단계. XP 누적으로 증가 |
| `character_xp` | `number` (≥ 0) | 누적 경험치. 운동·루틴 달성 시 획득 |

### 에너지 레벨 — 색상 결정 로직

캐릭터 몸 색상은 **수면과 루틴 점수의 평균**으로 결정됩니다.  
상태(workout/sleeping 등)와 완전히 **독립**적으로 동작합니다.

```
energy_score = (stats.sleep + stats.routine) / 2
energy_level = Math.min(6, Math.floor(energy_score * 7 / 100))
```

| energy_level | energy_score 범위 | 몸 색상 |
|:---:|---|---|
| 0 | 0 ~ 14.3 미만 | 거의 흰색 (에너지 없음) |
| 1 | 14.3 ~ 28.6 미만 | 옅은 하늘 (매우 부족) |
| 2 | 28.6 ~ 42.9 미만 | 하늘색 (부족) |
| 3 | 42.9 ~ 57.1 미만 | 옅은 보라 **← 기본** |
| 4 | 57.1 ~ 71.4 미만 | 살몬/분홍 (좋음) |
| 5 | 71.4 ~ 85.7 미만 | 주황/붉은 (높음) |
| 6 | 85.7 ~ 100 | 골드/노랑 (최고) |

| energy_level | 몸 색상 | 의미 |
|:---:|--------|------|
| 0 | 거의 흰색 | 수면·루틴 모두 극히 부족 |
| 1 | 옅은 하늘 | 매우 부족 |
| 2 | 하늘색 | 에너지 부족 |
| 3 | 옅은 보라 *(기본)* | 보통 상태 |
| 4 | 살몬/분홍 | 에너지 좋음 |
| 5 | 주황/붉은 | 에너지 높음 |
| 6 | 골드/노랑 | 최고 에너지 |

> **설계 원칙**: 몸 색상(에너지)과 표정·모션(상태)은 독립적입니다.  
> 예: 운동 중(workout) 상태이면서 에너지가 낮으면 하늘색 몸 + 운동 중 표정으로 렌더링됩니다.

**프론트엔드 계산 예시** (백엔드는 raw stats만 내려주면 됨):
```ts
const energyScore = (stats.sleep + stats.routine) / 2;
const energyLevel = Math.min(6, Math.floor(energyScore * 7 / 100));
```

### `workout_part` (선택)

가장 최근에 집중한 운동 부위. 해당 부위에 맞는 체형으로 캐릭터가 표현됩니다.

| 값 | 의미 | 체형 변화 |
|----|------|----------|
| `"chest"` | 가슴 운동 | 전반적으로 둥글고 가슴 범프가 풍성 |
| `"back"` | 등 운동 | 극도로 넓은 등판, 사각형 실루엣 |
| `"legs"` | 하체 운동 | 일반 상체 + 거대한 허벅지·발 |
| `"shoulders"` | 어깨 운동 | 어깨 범프가 극도로 넓고 두드러짐 |
| `"arms"` | 팔 운동 | 오른팔을 들어올려 이두 표현 (비대칭) |
| `"core"` | 복근 운동 | 납작한 직사각형 몸통 + 복근 라인 |

> 필드 미포함 시 기본 체형(NORMAL)으로 렌더링됩니다.  
> `sedentary`(운동 부족) · `overfed`(식단 불균형) 상태는 workout_part보다 우선합니다.

## 캐릭터 상태 결정 로직

프론트엔드가 응답을 받아 아래 우선순위로 캐릭터 상태를 결정합니다.  
백엔드는 **원시 데이터만** 내려주면 됩니다.

```
1순위  workout_status === 'active'    → workout  (운동 중)
2순위  is_sleeping === true           → sleeping (수면 중)
3순위  workout_status === 'completed' → flex     (운동 완료)

4순위  stats 중 가장 낮은 점수가 40 미만인 기둥:
       sleep    < 40  → tired      (수면 부족)
       exercise < 40  → sedentary  (운동 부족)
       diet     < 40  → overfed    (식단 불균형)
       routine  < 40  → derailed   (루틴 깨짐)
       (동점 시 위 순서가 우선)

5순위  (sleep + exercise + diet + routine) / 4 >= 75  → happy
6순위  그 외                                          → neutral
```

---

## 상태별 임계값 요약

| 상수 | 값 | 용도 |
|------|----|------|
| `SCORE_CRITICAL` | 40 | 기둥 점수가 이 값 미만이면 부족 상태 발동 |
| `SCORE_GREAT` | 75 | 전체 평균이 이 값 이상이면 `happy` 상태 |

---

## 폴링 주기 권장

| 상황 | 권장 주기 |
|------|----------|
| 일반 | 5분 |
| `workout_status === 'active'` 중 | 30초 |
| `is_sleeping === true` 중 | 폴링 불필요 (취침 종료 이벤트로 처리) |
