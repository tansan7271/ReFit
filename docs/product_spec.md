## 1. 한 줄 정의

> **"운동 중 기록은 많은 앱이 해줘. ReFit은 운동 전에 어떻게 준비하고, 운동 후에 어떻게 회복할지를 AI가 알려줘."**

- **앱 이름:** ReFit (_Recover your healthy workout routine_)
- **타겟 유저:** 운동 습관을 만들고 싶은 20-30대. 특히 비교·경쟁에 지쳐있고, 운동 전후 관리를 잘 모르는 사람
- **핵심 문제:** 운동 직전 뭘 먹어야 하는지, 운동 후 어떻게 회복해야 하는지 알려주는 앱이 없다

---

## 2. 핵심 차별점 (Why Us)

| 기존 앱                             | ReFit                                         |
| ----------------------------------- | --------------------------------------------- |
| 운동 **중** 기록 (세트, 무게, 거리) | 운동 **전/후** 가이드                         |
| 숫자·그래프 중심                    | 캐릭터가 내 상태를 대변                       |
| 랭킹·비교                           | 경쟁 없음, 나의 성장만                        |
| 매일 입력 요구                      | AI가 먼저 컨텍스트 파악 (날씨/수면 자동 수집) |

## 3. 기술 스택 (확정)

| 영역                      | 기술                                                       |
| ------------------------- | ---------------------------------------------------------- |
| **Frontend**              | React Native + **Expo (EAS Build)**                        |
| **Backend**               | FastAPI + MySQL + Alembic                                  |
| **AI**                    | Google Gemini API                                          |
| **Push Notification**     | FCM (Firebase Cloud Messaging) — 서비스 구현됨             |
| **날씨**                  | OpenWeather API                                            |
| **헬스 데이터 (iOS)**     | `react-native-health` — Apple HealthKit (수면, 걸음, 심박) |
| **헬스 데이터 (Android)** | `react-native-health-connect` — Google Health Connect      |
| **애니메이션**            | Lottie (캐릭터 상태 표현)                                  |
| **디자인**                | Figma                                                      |

> **Expo EAS Build 선택 이유:** 바이브 코딩에 최적화된 DX, 공통 코드 최대화,
> HealthKit/Health Connect 같은 네이티브 모듈은 EAS 커스텀 플러그인으로 처리.
> Managed Expo는 네이티브 모듈 불가 → bare/EAS 필수.

### 헬스 데이터 수집 전략

```
공통 인터페이스 (useHealthData hook)
    ├── iOS  → react-native-health (HealthKit)
    └── Android → react-native-health-connect (Health Connect)

수집 항목: 수면 시간, 걸음 수, 심박수(선택)
폴백: 플랫폼 연동 실패 시 유저 수동 입력 (수면 시간만)
```

## 4. MVP 스코프 (1달, 4인 팀, 바이브 코딩 기준)

### ✅ MVP에 포함 (반드시 완성)

#### 인증

- 회원가입 / 로그인

#### 온보딩 (첫 진입 시 1회)

- 운동 유형 선택 (헬스 / 러닝)
- 요일별 운동 루틴 설정
- 수면 목표 시간 설정
- 기본 신체 정보 (키, 몸무게, 숙련도)
- 캐릭터 소개 (이모지 기반, 1단계)

#### 홈 / 대시보드

- 오늘의 캐릭터 상태 표시 (수면 부족 / 정상 / 운동 완료 — 3가지)
- 오늘 운동 계획 요약
- 운동 시작 버튼

#### 운동 전 (Pre-workout)

- 아침 스마트 푸시 알림 (Gemini + 날씨 + 수면 기반 메시지 생성)
- 알림 탭 시 오늘의 준비 가이드 화면 (AI 생성 텍스트)

#### 운동 중 (In-workout)

- 세트 카운터 (웨이트) / 타이머 (러닝)
- 하단에 캐릭터 달리기 애니메이션 (Lottie 루프 1개)

#### 운동 후 (Post-workout)

- 세션 요약 카드 (시간, 세트 수, 소모 칼로리)
- Gemini 기반 회복 추천 메시지 (스트레칭, 수분, 영양)
- 오늘 수행 데이터 → 내일 알림 로직에 자동 반영

#### 마이페이지

- 뱃지 목록 (획득한 것 / 잠긴 것)
- 인바디 수치 수동 입력 탭
- 알림 시간 설정

---

## 10. 확정된 아키텍처 결정

### 헬스 권한 요청 시점
**온보딩 중 즉시 요청** — 중간에 권한 팝업이 뜨는 UX 마찰 방지.

### 알림 스케줄러: Expo 로컬 알림 방식 (확정)

실서버 없이 동작하는 방식:

```
앱 실행 시 (포그라운드 or 백그라운드 fetch)
  1. OpenWeather API → 내일 날씨 수집
  2. HealthKit / Health Connect → 오늘 수면 데이터 읽기
  3. Gemini API → 컨텍스트 기반 알림 메시지 생성
  4. expo-notifications → 유저 설정 시간에 로컬 알림 예약
```

- 서버 불필요, iOS/Android 동일하게 작동
- 앱을 오래 안 열면 내용이 오래될 수 있음 (과제 수준에서는 허용 범위)
- 나중에 실서버 붙일 때 FCM으로 쉽게 전환 가능하도록 알림 생성 로직은 서비스 레이어로 분리

**사용 패키지:** `expo-notifications`, `expo-background-fetch`, `expo-task-manager`
